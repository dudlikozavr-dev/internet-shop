# Mia-Amore Web Store — Спецификация

**Дата:** 2026-04-22
**Версия:** 2.0 (переписана после повторного Discovery)
**Статус:** Discovery Complete

---

## Executive Summary

Веб-магазин carstvo-sna.ru (поддомен) — это web-frontend для того же каталога, что продаётся через Telegram Mini App @sikretsweet_home_bot. Цель: расширить каналы продаж для физически имеющегося стока (лежит на балконе — остатки Mia-Amore, возвраты с InSales, WB/Ozon, товары других брендов). **Товары, цены и остатки общие с mini-app** — это единая платформа с двумя клиентами (Telegram и web). Трафик: email-рассылки, соцсети, SEO, ссылка из бота.

---

## Problem Statement

**Текущая ситуация:**
- Физический сток (Mia-Amore + остатки + чужие бренды) продаётся только через Telegram Mini App
- Mini-app ограничен Telegram-аудиторией
- Нет органического трафика, нет email-канала, нет web-точки входа

**Ограничения:**
- Люди, которые ищут в Яндексе/Google, не находят магазин
- Почтовая рассылка не работает (некуда вести клиента)
- Соцсети не работают (нельзя привязать ссылку для неактивированных пользователей Telegram)

**Решение:**
Веб-сайт с тем же каталогом, что и mini-app, на поддомене carstvo-sna.ru. Одна БД, один API, два фронта.

---

## Success Criteria

1. **Функциональность P0:** каталог, карточка, корзина, гостевой чекаут, оплата ЮKassa, email-подтверждение работают
2. **Синхронизация стока:** веб и mini-app читают/пишут в одну БД (timeweb PostgreSQL)
3. **SEO-ready:** Open Graph, sitemap.xml, meta-теги, SSR через Next.js
4. **152-ФЗ:** все персональные данные хранятся в РФ (timeweb VPS)
5. **Email-маркетинг:** интеграция с Unisender/Mindbox для рассылок
6. **Аналитика:** Яндекс.Метрика подключена с целями (просмотр/корзина/оплата)

## User Personas

### Мария, 30-45 — основной сегмент
- **Кто:** работающая женщина, пользуется email и соцсетями
- **Как находит:** почтовая рассылка, соцсети, поиск Яндекс
- **Что ожидает:** красивые фото, простой чекаут, доставка по РФ
- **Платит:** картой, СБП или QR
- **Девайс:** мобильный 60-70%

## User Journey

### 1. Первый визит
Источники: email-рассылка / соцсеть / Яндекс / ссылка из mini-app бота.

На главной сразу видит:
- Hero-баннер (красивое фото + заголовок + кнопка «Смотреть всё»)
- Скролл вниз → подборка «Новинки» / «Популярное»
- Шапка: логотип, корзина (со счётчиком)

### 2. Просмотр каталога
- Сетка товаров (фото + название + цена)
- Фильтры: категория, размер, цвет, диапазон цены
- Сортировка: новизна, цена ↑↓
- **Стоковый статус не показывается** (внутренне: если sold out — товар скрыт)

### 3. Карточка товара
- Галерея фото (4+)
- Название, цена
- Выбор размера и цвета
- Описание, материал, уход
- Кнопка «В корзину»

### 4. Корзина
- Список товаров, количество, сумма
- Возможность изменить количество/удалить
- Сохранение в localStorage

### 5. Оформление заказа (гостем)
- Форма: имя, телефон, email, адрес
- Выбор способа доставки:
  - Курьер по Москве
  - СДЭК **с выбором ПВЗ на интерактивной карте**
  - Почта России (ручной ввод адреса)
  - Яндекс.Доставка (ручной ввод адреса)
- Выбор способа оплаты:
  - Картой онлайн (ЮKassa)
  - СБП (ЮKassa)
  - QR-код (ЮKassa)
  - Наличные/терминал курьеру (только Москва, cash on delivery)
- Для онлайн-оплат → редирект на ЮKassa → возврат на «Спасибо!»
- Email с подтверждением

### 6. После покупки
- Для гостя: ссылка на статус заказа приходит на email (magic link, без пароля)
- Для зарегистрированного: личный кабинет с историей

## Technical Architecture

### Стек (подтверждён)

```
Frontend:    Next.js 14 (App Router, TypeScript, Tailwind CSS)
Backend:     FastAPI (расширение существующего backend mini-app)
Database:    PostgreSQL на timeweb VPS (уже мигрирован с Supabase → 152-ФЗ ✓)
Hosting:     timeweb VPS (один для mini-app + web)
Payments:    ЮKassa (карта + СБП + QR)
Email:       Unisender или Mindbox
Storage:     Cloudinary (фото — как у mini-app)
Analytics:   Яндекс.Метрика
```

### Схема системы

```
         ┌─────────────────┐            ┌─────────────────┐
         │  Telegram Mini  │            │    Web Browser  │
         │      App        │            │   (Next.js SSR) │
         └────────┬────────┘            └────────┬────────┘
                  │                              │
                  └──────────┬───────────────────┘
                             │
                             ▼
                   ┌───────────────────┐
                   │  FastAPI Backend  │
                   │   (timeweb VPS)   │
                   │                   │
                   │  /api/products    │  ← общие эндпоинты
                   │  /api/categories  │
                   │  /api/orders/web  │  ← уже есть, web клиент
                   │  /api/orders/tg   │  ← уже есть, mini-app
                   │  /api/payments/*  │  ← новые: ЮKassa webhook
                   │  /admin/*         │  ← существующие + фильтр источника
                   └─────────┬─────────┘
                             │
                             ▼
                   ┌───────────────────┐
                   │   PostgreSQL      │
                   │  (timeweb VPS)    │
                   │                   │
                   │  products         │  ← общие для web/mini-app
                   │  orders           │  ← поле source: 'web' | 'tg'
                   │  order_items      │
                   │  payments         │  ← новая: ЮKassa платежи
                   │  buyers           │  ← telegram_id nullable для web
                   └───────────────────┘
```

### Data Model (дополнения к существующей БД)

Существующие таблицы mini-app уже покрывают большую часть. Новые/дополненные поля:

```sql
-- Расширение существующей buyers:
ALTER TABLE buyers ADD COLUMN email VARCHAR(255);
ALTER TABLE buyers ADD COLUMN source VARCHAR(20) DEFAULT 'tg';  -- 'tg' | 'web'
-- telegram_id становится nullable для веб-покупателей

-- Расширение orders:
ALTER TABLE orders ADD COLUMN source VARCHAR(20) DEFAULT 'tg';
ALTER TABLE orders ADD COLUMN delivery_address JSONB;  -- для web: адрес/ПВЗ

-- Новая таблица payments (для ЮKassa):
CREATE TABLE payments (
  id SERIAL PRIMARY KEY,
  order_id INT REFERENCES orders(id),
  yookassa_payment_id VARCHAR(255) UNIQUE,
  amount DECIMAL(10,2),
  currency VARCHAR(3) DEFAULT 'RUB',
  method VARCHAR(50),        -- 'bank_card' | 'sbp' | 'qr' | 'cash_on_delivery'
  status VARCHAR(50),        -- 'pending' | 'succeeded' | 'canceled'
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

Новой отдельной БД **не нужно** — всё расширение существующей.

### Интеграции

#### ЮKassa (карта + СБП + QR)
- Одна интеграция покрывает 3 способа оплаты
- Flow: frontend → backend создаёт payment → redirect → webhook → статус

#### CDEK API
- Public endpoint для списка ПВЗ
- POST /calculator для расчёта стоимости
- Виджет на странице чекаута

#### Unisender/Mindbox
- Транзакционные письма (подтверждение заказа, статус)
- На P2: маркетинговые рассылки

#### Яндекс.Метрика
- Счётчик на всех страницах
- Цели: view_product, add_to_cart, begin_checkout, purchase

### Security Model

- **Гостевой чекаут:** без пароля, только валидация email/телефона
- **Admin auth:** существующий Bearer-token механизм mini-app (переиспользуется)
- **152-ФЗ:** данные в РФ (timeweb VPS)
- **HTTPS:** обязательно, Let's Encrypt
- **PCI DSS:** не требуется (ЮKassa хранит карты)
- **CSRF:** Next.js + token на форме
- **Rate limiting:** на чекаут и вход в admin

## Non-Functional Requirements

- **Performance:** Page Load < 3s на 3G, API < 500ms p95
- **Availability:** 99% uptime (MVP; timeweb VPS)
- **Backup:** ежедневный автоматический snapshot PostgreSQL
- **Scale (стартово):** неизвестен, дизайним под 100-500 посетителей/день. Сплеск до 1000/ч во время рассылки выдержим на Next.js (SSR кэшируется)
- **SEO:** SSR + sitemap + Open Graph
- **Mobile:** responsive, 60-70% трафика с мобильных

## Отложенные решения (не на MVP, но зафиксированы)

1. **Race condition на сток:** если один товар окажется в корзине и mini-app и web одновременно — пока не обрабатываем. Trafик низкий, проблема ~нулевая.
2. **Регистрация и личный кабинет:** только гостевой чекаут на MVP; аккаунты — P1.
3. **Статус «осталось 1 шт»:** скрыт на MVP. Внутренне: если sold out — товар скрывается.
4. **SMS:** на P1, после запуска базовой версии.
